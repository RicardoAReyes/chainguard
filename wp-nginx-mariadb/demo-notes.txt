  Ghost specifically — likely none. Here's why:

  packages.cgr.dev/extras contains Chainguard-curated app packages. Looking at what the WordPress apko.yaml pulls from there:

  ┌──────────────────────────┬────────────────────────────────────────────────────────────────┐
  │         Package          │                           What it is                           │
  ├──────────────────────────┼────────────────────────────────────────────────────────────────┤
  │ wordpress                │ WordPress core files, pre-packaged by Chainguard               │
  ├──────────────────────────┼────────────────────────────────────────────────────────────────┤
  │ wordpress-oci-entrypoint │ Chainguard's custom entrypoint script for WordPress containers │
  ├──────────────────────────┼────────────────────────────────────────────────────────────────┤
  │ wp-cli                   │ WP-CLI tool, re-packaged by Chainguard                         │
  └──────────────────────────┴────────────────────────────────────────────────────────────────┘

  These exist because Chainguard has specifically built and maintains WordPress as a product offering.

  Ghost is not a Chainguard product — there's no ghost or ghost-oci-entrypoint package in extras. Everything Ghost needs is either:

  ┌──────────────────────────┬──────────────────────────────────────────────────────────────────────┐
  │        Dependency        │                                Source                                │
  ├──────────────────────────┼──────────────────────────────────────────────────────────────────────┤
  │ nodejs-22, npm           │ packages.wolfi.dev/os (public)                                       │
  ├──────────────────────────┼──────────────────────────────────────────────────────────────────────┤
  │ sqlite-libs              │ packages.wolfi.dev/os (public)                                       │
  ├──────────────────────────┼──────────────────────────────────────────────────────────────────────┤
  │ build-base, python3, gcc │ packages.wolfi.dev/os (public, needed in melange for native modules) │
  ├──────────────────────────┼──────────────────────────────────────────────────────────────────────┤
  │ Ghost itself             │ npm (npm install ghost-cli) — goes into the melange recipe           │
  └──────────────────────────┴──────────────────────────────────────────────────────────────────────┘

  So the Ghost apko.yaml stays pointing only at packages.wolfi.dev/os. You'd only add packages.cgr.dev/extras if Chainguard were to officially package Ghost
  there in the future — or if you packaged it yourself and hosted it on a private extras-style repo.

  Pipeline steps:

  ┌─────────────────────┬────────────────────────────────────────────────────────────────────────────────────────────┐
  │        Step         │                                        What it does                                        │

  └─────────────────────┴────────────────────────────────────────────────────────────────────────────────────────────┘
